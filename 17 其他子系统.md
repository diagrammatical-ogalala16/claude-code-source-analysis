## 1. Vim 模式（5个文件）

### 1.1 完整状态机

Claude Code 实现了一个接近完整的 Vim 编辑器体验，核心是一个类型驱动的状态机：

```typescript
type VimMode = 'INSERT' | 'NORMAL';

type CommandState =
  | { type: 'IDLE' }                           // 等待命令
  | { type: 'OPERATOR_PENDING'; operator: Operator }  // 等待运动
  | { type: 'COUNTING'; digits: string }       // 数字前缀
  | { type: 'FIND_CHAR'; direction: 'f' | 'F' | 't' | 'T' }  // 查找字符
  | { type: 'REPLACE_CHAR' }                   // 替换字符
  | { type: 'MARK_SET' }                       // 设置标记
  | { type: 'MARK_JUMP' }                      // 跳转标记
  | { type: 'REGISTER_SELECT' }                // 选择寄存器
  | { type: 'SEARCH'; direction: '/' | '?' }   // 搜索模式
  | { type: 'COMMAND_LINE'; buffer: string }    // 命令行模式
  | { type: 'VISUAL'; start: Position }         // 可视模式
```

### 1.5 Dot-Repeat（点重复）

```typescript
interface RecordedChange {
  operator: Operator;
  motion?: Motion;
  textObject?: TextObject;
  count?: number;
  insertedText?: string;    // INSERT模式中输入的文本
}

// 记录变更
function recordChange(change: RecordedChange) {
  lastChange = change;
}

// 点重复
function dotRepeat(state: VimState): VimState {
  if (!lastChange) return state;
  return applyChange(state, lastChange);
}
```

### 1.6 寄存器（Register）

```typescript
interface VimRegister {
  content: string;
  type: 'linewise' | 'charwise';
}

// 粘贴行为根据类型不同
function paste(register: VimRegister, position: Position): void {
  if (register.type === 'linewise') {
    // 在当前行下方插入新行
    insertLineBelow(position.line, register.content);
  } else {
    // 在光标位置后插入
    insertAt(position, register.content);
  }
}
```

```typescript
interface PersistentVimState {
  lastChange: RecordedChange | null;  // 用于.重复
```

这些状态跨会话持久化到 `~/.claude/vim-state.json`。

### 为什么实现完整 Vim

目标用户群中有大量 Vim 用户。半成品的 Vim 模拟（只有 hjkl）反而会让 Vim 用户更痛苦——肌肉记忆触发了不存在的命令。所以要么不做，要么做完整。

### 为什么用类型驱动状态机

11 种 CommandState 的联合类型确保了：
- 每种状态转换都被穷举处理（TypeScript 的 exhaustive check）
- 不可能进入非法状态（类型系统阻止）
- 新增状态时编译器强制处理所有转换

```typescript
function generateBuddy(userId: string): Buddy {

  const hash = createHash('sha256').update(userId).digest();

  return {
    species: SPECIES[hash[0] % 18],        // 18种物种
    rarity: RARITIES[hash[1] % 5],         // 5种稀有度
    eyes: EYES[hash[2] % 6],               // 6种眼睛
    hat: HATS[hash[3] % 8],                // 8种帽子
    shiny: hash[4] < 13,                    // ~5%闪亮概率
    stats: {

      energy: (hash[6] % 5) + 1,
      curiosity: (hash[7] % 5) + 1,
      loyalty: (hash[8] % 5) + 1,
      mischief: (hash[9] % 5) + 1,
    },
  };
}
```

## 3. 语音输入（1个文件）

### 3.1 三层检查

```typescript
function isVoiceModeEnabled(): boolean {
  // Layer 1: 功能认证
  const hasVoiceAuth = isAuthenticatedWithAnthropicOAuth();
  if (!hasVoiceAuth) return false;

  // Layer 2: Provider限制
  if (isBedrock() || isVertex()) return false;

  // Layer 3: Kill-switch
  const disabled = checkGate_CACHED_OR_BLOCKING('tengu_amber_quartz_disabled');
  if (disabled) return false;

  return true;
}
```

### 为什么需要 OAuth 认证

语音识别需要 Anthropic 的后端服务支持。API key 用户和第三方 Provider（Bedrock/Vertex）无法访问语音服务。

### Kill-switch

`tengu_amber_quartz_disabled` 是一个服务端控制的开关。如果语音服务出现问题（质量下降、安全漏洞），可以在不发布更新的情况下立即禁用所有用户的语音功能。

## 4. 远程会话（4个文件）

### 4.1 RemoteSessionManager

```typescript
class RemoteSessionManager {
  private ws: WebSocket;
  private reconnectAttempt = 0;

  async connect(sessionUrl: string): Promise<void> {
    this.ws = new WebSocket(sessionUrl);

    this.ws.on('message', (data) => {
      const message = JSON.parse(data) as SDKMessage;
      this.handleMessage(message);
    });

    this.ws.on('close', () => {
      this.attemptReconnect();
    });
  }

  private async attemptReconnect(): Promise<void> {
    const delay = Math.min(
      1000 * Math.pow(2, this.reconnectAttempt),
      30000
    );
    this.reconnectAttempt++;

    await sleep(delay);
    await this.connect(this.sessionUrl);
  }
}
```

### 4.2 消息类型

```typescript
type SDKMessage =
  | { type: 'input'; content: string }
  | { type: 'tool_result'; toolUseId: string; result: any }
  | { type: 'abort' }
  | { type: 'ping' };

type ControlRequest =
  | { type: 'permission_request'; tool: string; input: any }
```

### 4.3 Viewer-Only 模式

```typescript
class RemoteViewer {
  private readonly readOnly: boolean;

  handleInput(message: SDKMessage): void {
    if (this.readOnly && message.type !== 'ping') {
      // Viewer只能ping（保活），不能发送任何输入
      return;
    }

    this.forwardToSession(message);
  }
}
```

Viewer-only 模式允许管理员或同事监控 Agent 的工作，但不能控制执行。这是远程会话的安全功能。

```typescript
  return [
    'Dream',
    'MonitorMcp',
  ].includes(task.type);

// 背景任务不显示在主任务列表中
function getVisibleTasks(tasks: Task[]): Task[] {
```

### 6.3 任务 UI 组件

```typescript
function TaskItem({ task }: { task: Task }) {
  const statusIcon = {
    pending: '⏳',
    running: '🔄',
    completed: '✅',
    failed: '❌',
    cancelled: '🚫',
  }[task.status];
```

### 为什么7种类型用联合而非继承

联合类型相比类继承的优势：
- **穷举匹配**：`switch (task.type)` 会在忘记处理某种类型时报编译错误
- **无运行时开销**：纯类型，不需要 `instanceof` 检查
- **可序列化**：任务可以直接 JSON.stringify/parse

### 7.2 迁移示例

```typescript
// 002: Sonnet 4.5 → Sonnet 4.6
const migration002: Migration = {
  id: '002_sonnet_45_to_46',
  description: '更新 Sonnet 4.5 引用为 Sonnet 4.6',

  shouldRun(settings: Settings): boolean {
    const model = settings.model || '';
    return model.includes('sonnet-4-5') || model.includes('sonnet-4.5');
  },

  run(settings: Settings): Settings {
    return {
      ...settings,
      model: settings.model!
        .replace('sonnet-4-5', 'sonnet-4-6')
        .replace('sonnet-4.5', 'sonnet-4.6'),
    };
  },
};
```

### 7.3 迁移执行保障

```typescript
async function runMigrations(): Promise<void> {
  const applied = await getAppliedMigrations();

  for (const migration of ALL_MIGRATIONS) {
    if (applied.has(migration.id)) continue;

    try {
      if (migration.shouldRun(currentSettings)) {
        currentSettings = migration.run(currentSettings);
        await saveSettings(currentSettings);
      }
      await markApplied(migration.id);
    } catch (error) {
      logger.error(`Migration ${migration.id} failed:`, error);
      // 失败不阻塞启动，下次会重试
    }
  }
}
```

迁移失败不会阻塞应用启动——用户不应该因为一个配置迁移错误而无法使用工具。

## 8. 设计决策总结

| 子系统 | 关键决策 | 原因 |
|--------|---------|------|
| Vim | 类型驱动状态机 | 穷举处理所有命令序列 |
| Vim | 持久化lastChange/lastFind | 跨会话的.和;重复 |
| Buddy | userId哈希确定性 | 无需服务端存储 |
| Buddy | LLM生成灵魂 | 每个Buddy独特的个性 |
| Voice | 三层检查 | 认证+Provider+Kill-switch |
| Remote | Viewer-only | 安全监控不能控制 |
| Remote | 指数退避重连 | 避免服务器过载 |
| Server | 状态机生命周期 | 清晰的转换规则 |
| Tasks | 7种联合类型 | 类型安全过滤和渲染 |
| Migrations | 失败不阻塞 | 可用性优先 |
