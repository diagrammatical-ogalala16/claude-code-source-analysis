### 1.3 设备路径封锁

```typescript
const BLOCKED_DEVICE_PATHS = [
  '/dev/zero',          // 无限输出零字节
  '/dev/urandom',       // 无限随机输出
  '/dev/random',        // 同上
  '/proc/self/environ', // 包含所有环境变量（含API密钥）
  '/proc/self/mem',     // 进程内存
  '/dev/stdin',         // 可能阻塞
  '/dev/fd/',           // 文件描述符
];

function isBlockedPath(path: string): boolean {
  const resolved = fs.realpathSync(path);
  return BLOCKED_DEVICE_PATHS.some(blocked =>
    resolved === blocked || resolved.startsWith(blocked)
  );
}
```

#### 为什么封锁这些路径

- `/dev/zero`、`/dev/urandom`：产生无限输出，读取会导致 OOM
- `/proc/self/environ`：包含所有环境变量，可能含有 `ANTHROPIC_API_KEY`、数据库密码等
- `/proc/self/mem`：可以读取进程的完整内存空间
- `/dev/stdin`：可能导致程序阻塞等待输入

### 1.4 自动编码检测

```typescript
async function detectEncoding(filePath: string): Promise<string> {
  const buffer = await fs.readFile(filePath, { length: 8192 });

  // 检查BOM
  if (buffer[0] === 0xEF && buffer[1] === 0xBB && buffer[2] === 0xBF) return 'utf-8';
  if (buffer[0] === 0xFF && buffer[1] === 0xFE) return 'utf-16le';
  if (buffer[0] === 0xFE && buffer[1] === 0xFF) return 'utf-16be';

  // 启发式检测
  if (isValidUtf8(buffer)) return 'utf-8';
  return 'latin1'; // 回退
}
```

### 3.2 为什么用字符串替换而非 diff/patch

这是一个关键的设计决策。LLM 生成的 unified diff 有以下问题：
- **行号不精确**：LLM 经常错误计算行号，导致 patch 应用在错误位置
- **上下文不匹配**：diff 的上下文行可能因为 LLM 的"幻觉"而与实际文件不同
- **格式错误**：diff 格式有严格的语法要求（`@@` 标记、`+/-` 前缀），LLM 经常出错

字符串查找替换的优势：
- **无歧义**：精确匹配，要么找到要么找不到
- **自描述**：`old_string` 就是上下文，不需要额外的行号
- **容错**：即使文件其他部分变了，只要目标字符串还在就能工作

### 3.3 引号保留

```typescript
function preserveQuotes(oldStr: string, newStr: string, fileContent: string): string {
  // 如果old_string在文件中只出现一次，直接替换
  const index = fileContent.indexOf(oldStr);
  if (index === -1) {
    throw new Error(`old_string not found in file`);
  }

  // 检查是否有第二次出现
  const secondIndex = fileContent.indexOf(oldStr, index + 1);
  if (secondIndex !== -1 && !replaceAll) {
    throw new Error(
      `old_string found multiple times. Provide more context to make it unique, or use replace_all.`
    );
  }

  return fileContent.slice(0, index) + newStr + fileContent.slice(index + oldStr.length);
}
```

### 3.4 CRLF/LF 处理

```typescript
function normalizeLineEndings(content: string, originalEnding: 'crlf' | 'lf'): string {
  // 检测原始文件的行尾风格
  const hasCRLF = content.includes('\r\n');
  const hasLF = content.includes('\n') && !hasCRLF;

  // 保持原始风格
  if (originalEnding === 'crlf') {
    return content.replace(/(?<!\r)\n/g, '\r\n');
  }
  return content.replace(/\r\n/g, '\n');
}
```

```typescript
  path: z.string().optional().describe("搜索根目录"),
```

### 4.2 结果限制

```typescript
const MAX_GLOB_RESULTS = 100;

async function executeGlob(pattern: string, rootPath: string): Promise<string[]> {
  const results = await glob(pattern, {
    cwd: rootPath,
    nodir: true,
    dot: false,      // 不匹配隐藏文件
    ignore: ['node_modules/**', '.git/**'],
  });

  if (results.length > MAX_GLOB_RESULTS) {
    return results.slice(0, MAX_GLOB_RESULTS);
    // 同时返回总数，告知模型结果被截断
  }

  return results;
}
```

#### 为什么限制100个文件

- 防止 token 爆炸：100 个文件路径约 3K-5K tokens，合理范围内
- 超过 100 个结果通常意味着模式太宽泛，模型应该细化搜索
- 结果排序按修改时间（最新优先），最相关的文件优先返回

### 4.3 相对路径节省 Token

```typescript
function formatResults(results: string[], rootPath: string): string {
  // 使用相对路径而非绝对路径
  return results
    .map(r => path.relative(rootPath, r))
    .join('\n');
}
```

相对路径比绝对路径短很多（`src/index.ts` vs `/home/user/projects/myapp/src/index.ts`），在 100 个结果中节省可观的 token。

### 4.4 权限规则检查

GlobTool 虽然是只读的，但需要检查路径权限——某些目录可能被规则禁止访问。

### 5.2 ripgrep 后端

```typescript
async function executeGrep(input: GrepInput): Promise<string> {
  const args: string[] = ['--json']; // JSON输出用于结构化解析

  if (input.multiline) args.push('-U', '--multiline-dotall');
  if (input['-i']) args.push('-i');
  if (input.glob) args.push('--glob', input.glob);
  if (input.type) args.push('--type', input.type);
  if (input.context) args.push('-C', String(input.context));

  // VCS排除
  args.push('--glob', '!.git', '--glob', '!.svn', '--glob', '!.hg',
            '--glob', '!.bzr', '--glob', '!.jj');

  args.push(input.pattern);
  args.push(input.path || '.');

  const result = await spawnRipgrep(args);
  return formatOutput(result, input);
}
```

### 5.3 head_limit 默认 250 的原因

250 行约 5K-10K tokens，是一个平衡点：
- 足够模型理解搜索结果的模式和分布
- 不会因为大项目的广泛匹配而溢出上下文
- 模型可以通过 `offset` 参数翻页获取更多结果

### 5.4 输出模式

| 模式 | 说明 | 用途 |
|------|------|------|
| `content` | 显示匹配行及上下文 | 查看代码细节 |
| `files_with_matches` | 只显示匹配文件路径 | 快速定位文件 |
| `count` | 显示每个文件的匹配数 | 评估修改范围 |

## 8. ToolSearchTool

### 8.1 搜索算法

```typescript
function searchTools(query: string, allTools: Tool[]): SearchResult[] {
  // 精确匹配
  if (query.startsWith('select:')) {
    const names = query.slice(7).split(',');
    return allTools.filter(t => names.includes(t.name));
  }

  // 关键词搜索
  return allTools
    .map(tool => ({
      tool,
      score: calculateScore(query, tool),
    }))
    .filter(r => r.score > 0)
    .sort((a, b) => b.score - a.score);
}

function calculateScore(query: string, tool: Tool): number {
  let score = 0;
  const terms = query.toLowerCase().split(/\s+/);

  for (const term of terms) {
    if (tool.name.toLowerCase().includes(term)) score += 10;
    if (tool.description.toLowerCase().includes(term)) score += 5;
    if (tool.searchHint?.toLowerCase().includes(term)) score += 8;
    if (tool.searchKeywords?.some(k => k.includes(term))) score += 6;
  }

  // MCP服务器工具加权
  if (tool.source === 'mcp') score += 12;

  return score;
}
```

```typescript
async function editNotebook(input: NotebookEditInput): Promise<string> {
  const notebook = JSON.parse(await fs.readFile(input.notebook_path, 'utf-8'));

  switch (input.operation) {
    case 'add_cell':
      notebook.cells.splice(input.cell_index ?? notebook.cells.length, 0, {
        cell_type: input.cell_type || 'code',
        source: input.content?.split('\n').map(l => l + '\n') || [''],
        metadata: {},
        outputs: [],
      });
      break;

    case 'edit_cell':
      notebook.cells[input.cell_index!].source =
        input.content!.split('\n').map(l => l + '\n');
      break;

    case 'delete_cell':
      notebook.cells.splice(input.cell_index!, 1);
      break;

    case 'move_cell':
      const [cell] = notebook.cells.splice(input.cell_index!, 1);
      notebook.cells.splice(input.target_index!, 0, cell);
      break;
  }

  await fs.writeFile(input.notebook_path, JSON.stringify(notebook, null, 1));
  return `成功执行 ${input.operation}`;
}
```

NotebookEditTool 也是延迟加载工具——大多数编程会话不涉及 Jupyter Notebook。

## 10. 设计决策总结

| 工具 | 关键决策 | 原因 |
|------|---------|------|
| FileRead | 设备路径封锁 | 防止无限输出和敏感信息泄露 |
| FileWrite | 文件历史追踪 | 支持撤销和审计 |
| FileEdit | 字符串替换而非diff | LLM 生成精确字符串比 diff 更可靠 |
| Glob | 100文件限制 | 防止 token 爆炸 |
| Grep | head_limit=250 | 平衡信息量和 token 消耗 |
| WebFetch | HTML→Markdown | 去除无用的HTML标签，保留语义内容 |
| WebSearch | 最多8结果 | 足够提供上下文，不浪费 token |
| ToolSearch | 加权搜索 | 根据匹配位置给予不同权重 |
| NotebookEdit | Cell级操作 | 避免整个notebook的序列化/反序列化 |
