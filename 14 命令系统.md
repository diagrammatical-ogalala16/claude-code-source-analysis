# 第14章 命令系统

> **作者**: [Wang Yanshu](https://catyans.github.io)

> 本章分析 commands/ 目录（103个子目录），涵盖命令类型、注册机制、可用性过滤和模式特定命令集。

## 1. 三种命令类型

### 1.1 Prompt 命令

```typescript
interface PromptCommand {
  type: 'prompt';
  name: string;
  description: string;

  // 可选约束
  allowedTools?: string[];
  progressMessage?: string;

  // 生成发送给Claude的提示
  getPromptForCommand(args: string): string;
}
```

Prompt 命令是最常见的类型。它们将用户输入转换为一段提示文本，发送给 Claude 处理。例如 `/commit` 会生成"分析当前 git 变更并生成 commit message"的提示。

**适用场景**：大多数命令，特别是需要 Claude 智能处理的任务。

### 1.2 Local-JSX 命令

```typescript
interface LocalJSXCommand {
  type: 'local-jsx';
  name: string;
  description: string;

  // 惰性加载React组件
  load(): Promise<React.ComponentType>;
}
```

Local-JSX 命令渲染一个 React 组件直接在终端中显示。例如 `/config` 渲染配置编辑界面，`/tasks` 渲染任务管理面板。

**适用场景**：需要复杂交互 UI 的命令，如配置编辑器、任务列表、差异查看器。

### 1.3 Shell 命令

```typescript
interface ShellCommand {
  type: 'shell';
  name: string;
  description: string;

  // 直接执行的Shell命令
  shellCommand: string;
}
```

Shell 命令直接执行一个 shell 命令。较少使用，主要用于简单的系统操作。

**适用场景**：不需要 Claude 参与的直接系统操作。

## 2. 命令定义结构

```typescript
interface Command {
  // 基本信息
  type: 'prompt' | 'local-jsx' | 'shell';
  name: string;
  description: string;

  // 约束
  allowedTools?: string[];

  // UI
  progressMessage?: string;

  // 生命周期
  getPromptForCommand?: (args: string) => string;
  load?: () => Promise<React.ComponentType>;

  // 可用性
  isEnabled?: (context: CommandContext) => boolean;
  availability?: CommandAvailability;
}

type CommandAvailability =
  | 'all'             // 所有用户
  | 'claude-ai'       // claude.ai 订阅者
  | 'console'         // console.anthropic.com用户
  | 'first-party'     // 第一方API用户
  | 'ant-internal';   // Anthropic员工
```

## 3. 命令注册机制

### 3.1 COMMANDS 缓存数组

```typescript
let cachedCommands: Command[] | null = null;
let cachedCwd: string | null = null;

function getCOMMANDS(cwd: string): Command[] {
  if (cachedCommands && cachedCwd === cwd) {
    return cachedCommands;
  }

  cachedCommands = buildCommandList(cwd);
  cachedCwd = cwd;
  return cachedCommands;
}
```

### 3.2 loadAllCommands（并行加载）

```typescript
async function loadAllCommands(cwd: string): Promise<Command[]> {
  // 并行加载三个来源
  const [skillCommands, pluginCommands, workflowCommands] = await Promise.all([
    loadSkillCommands(cwd),    // Skill转换为Command
    loadPluginCommands(),       // 插件Skill转换为Command
    loadWorkflowCommands(cwd), // 工作流转换为Command
  ]);

  return [...skillCommands, ...pluginCommands, ...workflowCommands];
}
```

### 3.3 getCommands（过滤和去重）

```typescript
function getCommands(context: CommandContext): Command[] {
  const all = getCOMMANDS(context.cwd);

  return all
    // 可用性过滤
    .filter(cmd => meetsAvailabilityRequirement(cmd.availability, context))
    // 启用检查
    .filter(cmd => !cmd.isEnabled || cmd.isEnabled(context))
    // 去重
    .filter((cmd, index, arr) =>
      arr.findIndex(c => c.name === cmd.name) === index
    );
}
```

## 4. 可用性过滤

### 4.1 meetsAvailabilityRequirement

```typescript
function meetsAvailabilityRequirement(
  availability: CommandAvailability | undefined,
  context: CommandContext
): boolean {
  if (!availability || availability === 'all') return true;

  switch (availability) {
    case 'claude-ai':
      return context.isClaudeAiSubscriber;

    case 'console':
      return !context.isClaudeAiSubscriber && context.isFirstParty;

    case 'first-party':
      return context.isFirstParty;

    case 'ant-internal':
      return context.isAntInternal;
  }
}
```

### 4.2 为什么区分订阅者和非订阅者

某些命令需要后端支持（如 `/share` 需要会话共享服务），只有 claude.ai 订阅者有权使用。API-key 用户使用 `console` 级别的命令集。这种分级确保每类用户看到的都是可用的命令。

## 6. 模式特定命令过滤

### 6.1 远程安全命令

```typescript
const REMOTE_SAFE_COMMANDS = [
  'compact',
  'cost',
  'config',
  'doctor',
  'diff',
  'context',
  'tasks',
  'theme',
];
```

远程会话中只允许安全的、不会泄露本地信息的命令。如 `/share`（会暴露本地文件路径）被排除。

### 6.2 Bridge 安全命令

```typescript
const BRIDGE_SAFE_COMMANDS = [
  ...REMOTE_SAFE_COMMANDS,
  'commit',
  'review',
  'memory',
  'skills',
];
```

Bridge 模式（IDE 集成）允许更多命令，因为有 IDE 的安全上下文保护。

### 6.3 过滤逻辑

```typescript
function filterCommandsForMode(
  commands: Command[],
  mode: ExecutionMode
): Command[] {
  switch (mode) {
    case 'remote':
      return commands.filter(c => REMOTE_SAFE_COMMANDS.includes(c.name));
    case 'bridge':
      return commands.filter(c => BRIDGE_SAFE_COMMANDS.includes(c.name));
    default:
      return commands;
  }
}
```

## 7. 动态 Skill 命令集成

### 7.1 插入位置

```typescript
function insertSkillCommands(
  builtInCommands: Command[],
  skillCommands: Command[]
): Command[] {
  // 找到内置命令列表中的正确插入位置
  // 通常在核心命令之后、高级命令之前
  const insertIndex = builtInCommands.findIndex(c => c.name === 'ultraplan');

  const result = [...builtInCommands];
  result.splice(insertIndex, 0, ...skillCommands);
  return result;
}
```

### 7.2 去重

```typescript
function deduplicateAgainstBuiltins(
  skillCommands: Command[],
  builtInNames: Set<string>
): Command[] {
  return skillCommands.filter(sc => {
    if (builtInNames.has(sc.name)) {
      // 用户Skill同名时覆盖内置（已在上层处理）
      // 插件Skill同名时被丢弃
      return sc.source === 'user';
    }
    return true;
  });
}
```

## 8. 命令解析与执行

### 8.1 解析流程

```typescript
function parseCommandInput(input: string): ParsedCommand | null {
  // 检查是否以/开头
  if (!input.startsWith('/')) return null;

  // 提取命令名和参数
  const spaceIndex = input.indexOf(' ');
  const name = spaceIndex === -1
    ? input.slice(1)
    : input.slice(1, spaceIndex);
  const args = spaceIndex === -1
    ? ''
    : input.slice(spaceIndex + 1);

  // 查找匹配的命令
  const command = findCommand(name);
  if (!command) return null;

  return { command, args };
}
```

### 8.2 执行分发

```typescript
async function executeCommand(parsed: ParsedCommand, context: ExecutionContext): Promise<void> {
  switch (parsed.command.type) {
    case 'prompt': {
      const prompt = parsed.command.getPromptForCommand(parsed.args);
      // 将提示注入到消息队列
      context.injectMessage({ role: 'user', content: prompt });
      break;
    }

    case 'local-jsx': {
      const Component = await parsed.command.load();
      // 渲染React组件
      context.renderOverlay(<Component args={parsed.args} />);
      break;
    }

    case 'shell': {
      // 直接执行shell命令
      await exec(parsed.command.shellCommand);
      break;
    }
  }
}
```

### 8.3 模糊匹配

```typescript
function findCommand(name: string): Command | null {
  // 精确匹配
  const exact = commands.find(c => c.name === name);
  if (exact) return exact;

  // 别名匹配
  const alias = commands.find(c => c.aliases?.includes(name));
  if (alias) return alias;

  // 前缀匹配（只有一个匹配时）
  const prefixMatches = commands.filter(c => c.name.startsWith(name));
  if (prefixMatches.length === 1) return prefixMatches[0];

  return null;
}
```

前缀匹配允许用户输入缩写：`/com` 匹配 `/commit`（如果只有一个以 "com" 开头的命令）。

## 11. 设计决策总结

| 决策 | 原因 |
|------|------|
| 三种命令类型 | prompt灵活、local-jsx丰富UI、shell直接 |
| 并行加载 | skills + plugins + workflows 同时加载 |
| 可用性分级 | 不同用户类型看到不同可用命令 |
| 按cwd缓存 | 避免重复扫描skills目录 |
| 模式过滤白名单 | 远程/Bridge环境安全第一 |
| 前缀匹配 | 提升输入效率 |
| 惰性组件加载 | local-jsx组件按需加载，不影响启动 |
